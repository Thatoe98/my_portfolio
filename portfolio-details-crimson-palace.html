Project Overview

Crimson Palace WebOS is a full-stack restaurant operating system that unifies customer ordering, cashier operations, kitchen ticket workflows, table service notifications, management dashboards, and inventory visibility into one web platform.

The project models a realistic restaurant environment with seeded operational datasets (sales, ingredient usage, payroll, and stock snapshots) while supporting live transactions such as QR ordering, POS order entry, and kitchen state progression.

Core Modules

Manager Dashboard

Tracks revenue, gross and operating profit, menu performance, cost breakdowns, and high-priority inventory alerts in one executive view.

QR Ordering & POS

Supports direct table-based mobile ordering and staff-led POS workflows with shared menu and table models.

Kitchen Display System

Moves tickets across waiting, in-prep, ready, and served states with frequent polling for real-time operational visibility.

Inventory Oversight

Surfaces reorder risk, expiry pressure, and daily stock intelligence through alert-first inventory monitoring.

Technical Strengths

Workflow-Centered Architecture

  • Monolithic full-stack design with clear UI, API, and persistence layering
  • Operational modules mapped to real restaurant roles and routines
  • Consistent API success and error contracts for all clients
  • Selective state persistence based on workflow behavior

Database and SQL Procedures

  • Normalized schema covering lookup, master, fact, and runtime entities
  • Transactional order creation via create_customer_order procedure
  • Kitchen state transitions via set_kitchen_ticket_status procedure
  • Dedicated notification table and indexes for table service polling

Data Engineering Discipline

  • Dependency-aware seed pipeline from structured CSV sources
  • Normalization for dates, percentages, currency, and units
  • Validation pass for counts, ranges, and integrity assumptions
  • Reproducible time simulation for consistent analytics interpretation

Product Thinking

  • Designs for customers, kitchen staff, cashiers, and managers
  • Combines historical analytics with live transaction processing
  • Bridges front-of-house operations with stock risk management
  • Pragmatic tradeoffs: monolith deployment and polling-first realtime

Skills Demonstrated

  • Full-Stack Engineering: Unified development across app routes, APIs, and relational persistence.
  • Domain Modeling: Restaurant-specific schema design supporting both operations and analytics.
  • Database Reliability: SQL procedures for transactional correctness in critical workflows.
  • Data Pipeline Design: Structured ingestion and validation from spreadsheet-style sources.
  • System Design: Practical architecture decisions balancing speed, clarity, and operational value.
